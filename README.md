# User Activity API

API для анализа активности пользователей (тестовое задание)

## Исправления и допущения в ТЗ

В таблице `logs` отсутствует связь с постом (невозможно получить header и автора для датасета comments).
**Решение:** добавлена колонка `post_id` (Integer, nullable=True). При событии `comment` туда будет записываться id поста. Внешние ключи между разными БД не поддерживаются SQLite, поэтому связь обеспечивается на уровне приложения.

Все остальные требования выполнены точно по ТЗ.

## Запуск проекта

### Через Makefile
```bash
make run    # запустить проект
make stop   # остановить проект
make test   # запустить тесты
make seed   # заполнить БД тестовыми данными
make help   # показать все команды
```

### Через Docker
```bash
docker-compose up --build
```


## API Endpoints

После запуска сервер доступен по адресу: **http://localhost:8000**

| Endpoint | Метод | Параметр | Описание |
|----------|-------|----------|----------|
| `/api/comments` | `GET` | `login` | Датасет комментариев пользователя |
| `/api/general` | `GET` | `login` | Общая статистика по дням |
| `/docs` | `GET` | - | [Swagger документация](http://localhost:8000/docs) |



## Тестовые пользователи
Для проверки работы API заполнить базу тестовыми данными:

```bash
make seed
```

После заполнения БД тестовыми данными (`make seed`) доступны следующие логины:

- `Masha`
- `Kate`
- `Fedya`

Используйте их для проверки API
